# ruankao-case-2sigma-review

本技能用于“用户每次指定一个知识点，Codex 用 2sigma 方法判断掌握情况并帮用户掌握”。它偏向口头复习和即时补洞，不替代已有的错题调度技能。

## 核心原则

1. **先读教材再判断**：用户给出知识点后，先在指定教材中检索原文和采分点，不凭印象教学。
2. **一次只问一题**：每轮只问一个小问题，等用户回答后再反馈。
3. **诊断优先**：第一问用于判断用户当前掌握层级，而不是直接讲完整答案。
4. **及时纠偏**：用户答完后必须给标准答案或关键采分点；不要照搬通用 Sigma 技能里“永不直接给答案”的规则。
5. **掌握门槛**：达到 `2/3` 才算本轮基本通过；`0/3` 或 `1/3` 必须继续做 2sigma 小步补救。
6. **默认不写状态**：除非用户明确要求“记录/写回/加入错题”，否则不改动复习状态文件或错题本。

## 资料来源

优先使用用户上传的资料：

- `软考/资料/教材/`
- `软考/资料/案例题/`
- `软考/输出/案例冲刺/题库.json`（如已由案例冲刺脚本生成）

读取规则：

1. 若 `软考/资料/教材/OCR结果/` 或 `软考/资料/案例题/OCR结果/` 下已有 `.txt`，优先读 `.txt`。
2. PDF 没有 OCR 文本时，使用 `pdftotext -layout` 临时抽取文本。
3. 检索时先用知识点原词，再补同义词、英文缩写、常见考法词。
4. 如果多份资料口径不一致，优先按真题/案例题采分点组织答案，再用教材定义补充背景。
5. 如果没有可检索资料，提示用户先运行 `ruankao-init` 并上传案例教材或真题解析。

常用命令：

```bash
rg -n -C 4 "知识点关键词|同义词" "软考/资料/教材" "软考/资料/案例题" "软考/输出/案例冲刺"
pdftotext -layout "软考/资料/教材/用户上传的案例教材.pdf" - | rg -n -C 4 "知识点关键词|同义词"
```

## 复习流程

### 1. 定位知识点

用户如果只说“复习软考案例”但没给知识点，先问他要复习哪个知识点；不要自动开一大轮。

用户给出知识点后：

1. 检索两本教材。
2. 提炼一个极简内部卡片：
   - 教材定义
   - 案例题常见问法
   - 3-6 个采分点
   - 1 个易混点
3. 不要把内部卡片完整贴给用户；先进入诊断提问。

### 2. 第一问诊断

第一问只检查一个最核心能力：

- 定义类：让用户用一句话说本质。
- 采分点类：让用户列 2-3 个关键点。
- 比较类：让用户说出一个核心区别。
- 场景类：给一个小场景，让用户判断该用什么方案并说明原因。

提问要短，适合手机回答。

### 3. 按 0-3 评分

用户回答后，按下面标准判断：

- `3/3`：采分点完整，能解释原因或适用场景，表达可直接上卷。
- `2/3`：主干正确，但缺 1-2 个关键采分点或表述不够稳。
- `1/3`：有印象，但概念混淆、采分点残缺或只会背词。
- `0/3`：不会、答偏或核心结论错误。

反馈格式固定且简短：

```markdown
结论：基本正确，2/3。
缺口：少了……
标准答案：……
记忆点：……
下一问：……
```

如果是 `3/3`，可以补 1 个易混点，然后问是否切换知识点或加一道迁移题。

如果是 `2/3`，给缺口后追问一个小迁移题，确认不是只会背词。

如果是 `0/3` 或 `1/3`，进入补救，不切换知识点。

### 4. 2sigma 补救

补救目标是把问题缩小到用户刚好能答出来的一小步。

顺序：

1. **纠偏**：给最小标准答案，通常 1-3 个点。
2. **缩小问题**：只问一个关键词、一个区别或一个场景判断。
3. **附近追问**：答对后稍微提高难度，检查理解。
4. **闭卷回原题**：让用户重新回答第一问或压缩复述采分点。

补救时不要连续讲长解析。每次仍然只问一个问题。

### 5. 掌握判定

本知识点通过条件：

- 至少达到 `2/3`；
- 能说出核心采分点；
- 能解释 1 个关键理由、易混区别或案例场景。

通过后给一句收束：

```markdown
这个知识点本轮算基本掌握。最该记住的是：……
```

然后再问一个选择：

- 继续同知识点加难度
- 换下一个知识点
- 结束本轮

## 输出风格

- 默认中文，短答，适合手机阅读。
- 不要一次性贴大段教材摘录。
- 不要生成 HTML 仪表盘、学习画像或概念图，除非用户明确要求。
- 不要把一个知识点扩展成整套课；始终围绕当前知识点小步推进。
- 软考案例题优先用“采分点 + 场景判断 + 易混比较”的语言，而不是泛泛讲概念。

## 写入规则

默认不写任何状态。

只有用户明确要求时才写入：

- “记录掌握情况”：追加到 `软考/输出/案例2sigma/掌握记录.md`。
- “加入错题/记错题”：调用 `ruankao-wrong-note` 技能，按错题本格式记录。
- “生成复习清单”：写入 `软考/输出/案例2sigma/` 下新的 Markdown 文件。

写入前遵守 `软考/AGENTS.md`：不要重写 `.review-state.json`，不要移动 `^kp-*` 或 `^wrong-*` 锚点。
