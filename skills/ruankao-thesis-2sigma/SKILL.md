# ruankao-thesis-2sigma

本技能用于论文复习的短循环训练：先读取用户上传的范文和教材资料，再问 1 个回忆题，等用户回答后判断掌握程度，给出最小纠偏，并继续推进下一小步。

训练目标分三层：

1. **理解知识点**：知道论文题目在考什么、解决什么问题、教材中有哪些支撑概念。
2. **背诵范文骨架**：掌握可复用结构、关键段落和项目化表达，不要求逐字照背。
3. **母题迁移子题**：能把一篇母题迁移到相关子题，而不是只押单一题目。

三层不是三选一。一次训练通常会围绕同一母题逐步覆盖三层，只是难度按用户回答动态调整。

## 2-Sigma 规则

- **先诊断**：开始训练前先判断用户当前卡在哪一层：题眼、本质、三关键词、段落骨架、项目动作或迁移能力。
- **一次一问**：每轮只问 1 个问题，等用户回答后再继续。
- **掌握门槛**：当前层约 `80%` 稳定后再升难度；主旨正确、关键词基本齐、至少能连到一个项目动作，才算过关。
- **误区追踪**：回答不完整时，只标一个主要问题：`题眼偏移`、`教材概念薄`、`范文骨架断`、`项目动作空`、`迁移不会替换`、`表达不成段`。
- **间隔交织**：每 3-4 轮，把当前母题和一个旧母题或子题做小对比，避免只会顺序背。
- **校准信心**：当用户自信和答案质量不匹配时，问一句“这段你现在几成把握？”即可，不做问卷。
- **不乱造文件**：默认不创建学习画像、HTML 看板或概念图；除非用户明确要求。

## 资料读取顺序

训练前必须先读资料，不凭印象教学。

1. 范文来源：`软考/资料/论文范文/`
2. 教材与知识补充：`软考/资料/教材/`、`软考/资料/论文知识/`
3. 项目例子：只读 `软考/资料/项目背景/项目背景卡.md`
4. 已沉淀素材：`软考/输出/论文素材/`

常用检索命令：

```bash
rg -n "架构演化|服务网格|数据集成|系统测试" "软考/资料/论文范文" "软考/资料/论文知识" "软考/输出/论文素材"
python3 .agents/skills/ruankao-thesis-argument-prep/scripts/search_textbook_angles.py --vault /path/to/vault "架构演化" "可观测" "数据集成"
```

如果没有范文或项目背景卡，先提示用户运行 `ruankao-init` 上传资料或填写项目背景卡。

## 母题来源

母题来自用户上传的范文。没有固定内置押题。

常见可准备的母题类型包括：

- 系统测试：功能、性能、安全、兼容、可靠性测试
- ABSD：架构需求、架构设计、文档化、复审、实现、演化
- 架构演化：变化点识别、扩展设计、版本兼容、灰度回滚、治理机制
- 数据集成：ETL、消息同步、联邦查询、主题建模、质量血缘
- 服务网格：Sidecar、数据平面、控制平面、流量治理、安全通信、可观测
- 存储计算分离：存储独立、计算弹性、资源隔离、分层存储、可靠性

如果用户没有指定母题，优先从最近练过或最弱的母题继续；没有上下文时，从用户上传范文标题里给一个简短选择集。没有范文时，先请用户上传。

## 三层训练流程

每次只推进一小步，不要一次性讲完整论文。

对每个母题，按以下顺序训练：

1. **题眼定位**：让用户用一句话说清本质。
2. **教材扩充**：补 2-4 个教材角度，避免只背模板。
3. **三关键词**：把主体压缩成 3 个稳定关键词。
4. **分段背诵**：按 `摘要 -> 项目背景 -> 技术方法说明 -> 主体三段 -> 结尾` 逐段训练。
5. **项目映射**：把关键词和教材角度映射到用户项目背景卡。
6. **子题迁移**：把母题迁移到一个相关子题。
7. **理论段**：训练 100-150 字理论段。
8. **主体段**：训练 `原则/方法 + 项目动作 + 效果` 的正文段。
9. **闭卷回忆**：让用户闭卷回忆三关键词、一个段落骨架和一个迁移例子。

难度调节：

- 答得流畅准确：稍微升难度。
- 方向对但缺点：补缺口，再问一个邻近小题。
- 答错或太虚：缩小到一个关键词、一个区别或一个项目动作。
- 弱答后只给最小纠偏，不倒一整篇解析。

## 第一层：知识点理解

在背范文前，先确认用户真的理解题目。

最小检查：

- 用一句话说题眼本质。
- 说明它解决真实系统里的什么问题。
- 说出 3-5 个教材核心概念。
- 解释一个易混对比，例如：系统测试 vs 集成测试、服务网格 vs 微服务、数据集成 vs 接口聚合。

期望答案形态：

```markdown
本质：……
解决的问题：……
核心概念：A / B / C
易混点：……不是……，而是……
```

如果用户只能背词、不能解释，就继续停留在第一层。

过关标准：

- 能一句话说清本质。
- 能连接到一个真实系统问题。
- 能说出至少 3 个可用教材概念或质量属性。
- 能用自己的话讲清一个易混点。

## 第二层：范文骨架背诵

第一层基本正确后，训练范文作为可复用记忆材料。

固定结构：

- 摘要：项目 + 角色 + 题眼 + 本文论述
- 项目背景：项目规模 + 职责 + 业务链路 + 痛点
- 技术方法说明：定义 + 方法/原则 + 实施过程 + 适用性
- 主体三段：三关键词，每段都用 `论点 + 项目动作 + 效果`
- 结尾：成效 + 不足 + 后续改进 + 主题升华

背诵推进：

1. 先问段落关键词，不直接要求全文。
2. 再让用户从关键词恢复一句话。
3. 再让用户用自己的话写一小段。
4. 对照范文指出缺的采分元素。
5. 多个小段稳定后，才要求连续复述。

不要求逐字背诵，除非用户明确要求。默认追求“采分内容一致，表达自然”。

过关标准：

- 能不提示说出文章结构。
- 能说出主体三关键词。
- 能恢复一句理论句和一个主体段。
- 主体段必须有 `原则/方法 + 项目动作 + 效果`，空口号不过关。

## 第三层：母题迁移

范文骨架稳定后，训练迁移能力。

每个母题至少构建一张迁移图：

```markdown
母题：……
原始三关键词：……
教材补充点：……
可扩展子题目：……
沿用内容：……
替换内容：……
容易跑题点：……
```

典型问题：

- 如果题目从 `系统测试` 变成 `性能测试`，哪些段落骨架能沿用？哪些概念必须替换？
- 如果 `架构演化` 变成 `软件维护`，哪些项目动作还能用？
- 如果 `服务网格` 变成 `可观测架构`，哪一段可以变成主段？

过关标准：

- 能说出母题中可沿用的部分。
- 能说出子题中必须替换的部分。
- 能补一个教材支撑概念。
- 能给一个符合用户项目背景卡的项目动作。

## 迁移规则

常见迁移方向：

- 架构演化 -> 软件维护、可修改性、持续交付、DevOps、服务治理
- 系统测试 -> 性能测试、自动化测试、可靠性测试、质量保证
- 数据集成 -> 数据迁移、大数据、数据质量、主数据管理
- 服务网格 -> 微服务治理、可观测架构、流量治理、安全通信
- 存储计算分离 -> 云原生架构、大数据平台、资源隔离、成本优化
- ABSD -> 软件架构设计、架构评估、质量属性、构件复用

迁移时不要编造教材理论。先检索教材或用户笔记，再转述成考试可用表达。

## 反馈标准

默认短答，适合手机阅读。

用户回答后：

- 先判断：`正确` / `基本正确` / `不完整` / `错误`
- 给必要的标准答案，不长篇展开。
- 不满分时只指出 1 个主要误区。
- 给 1 个最该记住的记忆点。
- 最后只问 1 个下一题。

答对时可以顺手补 1 个易混点；答错时先给纠正答案，再问下一题。

## 训练目标

训练用户形成以下可复用能力：

- 12 字左右的主体结构，例如 `开闭解耦 / 契约兼容 / 灰度回滚`
- 一句目标句，例如 `通过……实现……`
- 100-150 字理论段
- 一个 `原则/方法 + 项目动作 + 效果` 主体段
- 一个母题迁移到子题的说明
- 一篇范文各段的回忆骨架

项目内容只使用用户项目背景卡：

- 项目名称：读取 `软考/资料/项目背景/项目背景卡.md`
- 本人角色：读取卡片，不编造
- 核心场景：只使用卡片列出的场景
- 成效：优先使用卡片已有成效；没有时用保守、可辩护表达

## 完成标准

不要因为用户“看过模板”就判定完成。

一个母题完成，必须三层都过关：

- 第一层：能说清本质，并补一个教材角度。
- 第二层：能回忆文章结构、三关键词、一个理论段、一个主体段。
- 第三层：能写一个项目段，并迁移到至少一个相关子题。

结束前做一次闭卷检查：

```markdown
请闭卷回答：这个母题的 3 个关键词、1 句理论句、1 个项目动作、1 个可迁移子题是什么？
```

最后给一个极小保持任务，例如：

```markdown
10 分钟后，闭卷回忆三关键词 + 一个理论句 + 一个子题迁移。
```

## 边界

- 用户未要求记录时，不修改笔记或复习状态。
- 不一次问多个复习题。
- 不过度押题；必须补教材扩展。
- 用户没要求完整作文时，不代写长篇全文。
- 不长篇引用 PDF 原文，只转述成考试表达。
