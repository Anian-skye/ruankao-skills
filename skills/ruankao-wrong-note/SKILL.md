# ruankao-wrong-note

本 Skill 用于记录软考错题，自动完成以下工作：
1. 按天记录（每天一个文件，支持多条错题）
2. 表格形式紧凑记录
3. 在知识点索引中建立精确双向链接（题目级别 ↔ 知识点级别）

## 使用方式

用户提供错题信息时，按以下格式组织内容：

```
## 错题信息
- 题目：xxx
- 题型：选择题 / 案例题
- 选项：A. xxx；B. xxx；C. xxx；D. xxx（选择题必填，案例题可省略）
- 我的答案：xxx
- 正确答案：xxx
- 错误原因：xxx
- 知识点：xxx
```

或者简洁描述：
- 题目、我的答案、正确答案、错误原因、涉及知识点

记录时必须标明题型：

- `综合题/选择题`：有 A/B/C/D 等选项，或答案以选项字母为主。必须尽量保留完整选项；如果用户没有给选项，先从截图/原题/上下文补齐，仍缺失时写 `选项：待补充`，不要把选择题改写成纯背诵题。
- `案例题`：简答、分析、设计、说明类题目，需要按采分点组织答案。
- 如果题型暂时无法判断，写 `题型：待确认`，并在解析或备注里说明缺什么信息。

## 执行流程

### 第一步：确定日期

使用当天的日期，格式：`软考/输出/错题本/YYYY-MM-DD.md`

如果 `软考/输出/知识点索引.md` 不存在，先创建最小索引或建议用户运行 `ruankao-init`。

- 如果文件已存在，追加新错题
- 如果是新文件，创建并添加

### 第二步：确定知识点

根据题目内容，在 `软考/输出/知识点索引.md` 中找到对应的**可归类复习的一层知识点**。不要只停留在章节级别，也不要细到几乎与题目标题一模一样。

**常见知识点映射**：
- PKI/数字证书 → 7.1 信息安全 → 身份认证
- MQTT/物联网协议 → 第八部分 → 物联网
- 边缘计算/分布式 → 6.2 分布式系统 → 边缘计算
- OSI/TCP/IP → 3.1 网络体系结构
- CAP/BASE → 6.2 分布式系统
- 流水线/CISC/RISC → 1.3 体系结构

**归类原则**：
- 优先链接到能串联同类题的知识点，而不是题目本身的“同义改写”
- 如果两道题本质属于同一类概念，尽量挂到同一个知识点下方便对比复习
- 例如：
  - Redis跳跃表、Redis Hash冲突 → `Redis数据结构与特性`
  - Redis 6.0多线程 → `Redis高性能原理`
  - RedLock算法 → `分布式锁（数据库、Redis、Zookeeper）`
  - 命令模式识别 → `设计模式`
  - UML序列图条件分支 → `UML建模（用例图、类图、序列图、状态图）`
  - MQTT协议 → `物联网`

**如果找不到对应知识点**：
- 在相关章节末尾添加新知识点
- 格式：`- [ ] 新知识点名称 ^kp-知识点ID`
- 同时添加精确反向链接：`- 错题：[[软考/输出/错题本/YYYY-MM-DD#^wrong-YYYY-MM-DD-NN|YYYY-MM-DD · 错题 N：标题]]`

### 第三步：生成/更新错题文件

使用以下模板格式：

```markdown
# 错题记录 - 2026年4月4日

| 序号 | 题目 | 题型 | 知识点 | 错误原因 |
|------|------|------|--------|----------|
| 1 | [边缘计算优势](#^wrong-2026-04-04-01) | 案例题 | [边缘计算](../知识点索引.md#^kp-edge-computing) | 知识点遗漏 |

---

### 错题 N：简短标题 ^wrong-YYYY-MM-DD-NN

| 题目 | 完整的题目内容 |
|------|------|
| 题型 | 选择题 / 案例题 |
| 选项 | A. xxx<br>B. xxx<br>C. xxx<br>D. xxx（选择题必填；案例题可写“无”） |
| 我的答案 | 当时选择的答案 |
| 正确答案 | 正确答案 |
| 错误原因 | 知识点遗漏 / 理解偏差 / 粗心大意 / 审题不清 / 计算错误 |
| 知识点 | [[软考/输出/知识点索引#^kp-知识点ID]] |
| 解析 | 详细解析内容 |
```

**关键点**：
- 多条错题之间用 `---` 分隔
- 每条错题都必须带独立 block id，格式为 `^wrong-YYYY-MM-DD-NN`
- **必须使用 Markdown 表格语法**，禁止使用 HTML `<table>` 标签
- 知识点链接必须使用 wikilink，并且直达**归类后的知识点 block id**，不能只链到章节，也不要细到和题目同名
- 日期页顶部总表中的题目列建议使用 Markdown 锚点链接，例如 `[题目标题](#^wrong-...)`
- 日期页顶部总表必须包含 `题型` 列；新增错题至少标明 `选择题` 或 `案例题`
- 日期页顶部总表中的知识点列建议使用 Markdown 链接，例如 `[知识点名称](../知识点索引.md#^kp-...)`
- 选择题必须保留 `选项` 行，正确答案应写成 `A：选项内容` 这类形式；如果原选项缺失，写 `选项：待补充` 并在解析中注明

### ⚠️ 重要提醒：必须使用 Markdown 表格样式

**必须使用以下 Markdown 表格格式，禁止使用 HTML 表格：**

```markdown
| 题目 | 完整的题目内容 |
|------|------|
| 题型 | 选择题 / 案例题 |
| 选项 | A. xxx<br>B. xxx<br>C. xxx<br>D. xxx（选择题必填；案例题可写“无”） |
| 我的答案 | 当时选择的答案 |
| 正确答案 | 正确答案 |
| 错误原因 | 知识点遗漏 / 理解偏差 / 粗心大意 / 审题不清 / 计算错误 |
| 知识点 | [[软考/输出/知识点索引#^kp-知识点ID]] |
| 解析 | 详细解析内容 |
```

**禁止使用如下 HTML 表格格式：**
```html
<table>
<tr><th>题目</th><td>...</td></tr>
<tr><th>我的答案</th><td>...</td></tr>
</table>
```

**原因**：Markdown 表格更通用，与 Obsidian 生态兼容性更好，且易于阅读和编辑。

### 第四步：更新知识点索引

在 `软考/输出/知识点索引.md` 中找到对应知识点的位置，添加错题链接：

```markdown
- [ ] 具体知识点 ^kp-知识点ID
  - 错题：[[软考/输出/错题本/YYYY-MM-DD#^wrong-YYYY-MM-DD-NN|YYYY-MM-DD · 错题 N：标题]]
```

**两种情况**：

1. **知识点已存在**：复用已有 `^kp-*` block id，在该知识点下追加精确到单题的错题链接
2. **知识点不存在**：
   - 在相关章节末尾添加新知识点
   - 格式：`- [ ] 新知识点名称 ^kp-知识点ID`
   - 同时添加错题链接：`- 错题：[[软考/输出/错题本/YYYY-MM-DD#^wrong-YYYY-MM-DD-NN|YYYY-MM-DD · 错题 N：标题]]`

**注意**：
- 保持原有格式和勾选框
- 如果知识点下已有错题链接，添加新的一条
- 如果该知识点已经形成一组同类题，优先补 1-2 行“同类记忆”或“易混点”，让索引兼具复习提纲作用
- 如果知识点不在索引中，**必须更新索引补充进去**，然后建立双向链接

## 示例

### 输入
用户提供了边缘计算的错题：

```
题目：从数据通信、数据安全和系统性能等方面分析边缘计算的优势
我的答案：不清楚
正确答案：数据通信更快、数据更少；数据更安全；性能更高
错误原因：知识点遗漏
```

### 执行

1. **确定日期**：2026-04-04
2. **确定知识点**：边缘计算 → 6.2 分布式系统
3. **更新错题文件**：在现有文件中追加新错题
4. **更新知识点索引**：在"边缘计算"下添加链接

### 输出

错题文件增加：
```markdown
### 错题 2：边缘计算优势 ^wrong-2026-04-04-02

| 题目 | ... |
|------|------|
| 我的答案 | ... |
...
```

知识点索引更新：
```markdown
- [ ] 边缘计算 ^kp-edge-computing
  - 错题：[[软考/输出/错题本/2026-04-04#^wrong-2026-04-04-02|2026-04-04 · 错题 2：边缘计算优势]]
```

### 知识点未找到时的处理

如果知识点索引中没有对应知识点，需要：

1. **在相关章节末尾添加新知识点**
2. **建立双向链接**

示例：假设有一道关于 "Kubernetes" 的错题，但索引中没有：

```markdown
### 第八部分：新技术与趋势

- [ ] 云计算
- [ ] 大数据
- [ ] 人工智能
- [ ] 物联网
- [ ] 容器编排（Kubernetes） ^kp-kubernetes
  - 错题：[[软考/输出/错题本/2026-04-05#^wrong-2026-04-05-01|2026-04-05 · 错题 1：标题]]
- [ ] 区块链
```

## 注意事项

- **必须确保知识点索引中有对应知识点**：记录错题前，必须先检查 `软考/输出/知识点索引.md` 中是否存在该知识点。如果不存在，**必须先添加到索引中**，然后再建立双向链接。禁止在索引中没有对应知识点的情况下强行记录错题。
- 一天可以记录多条错题，用 `---` 分隔
- 表格使用 Markdown 表格语法，确保格式统一
- 双向链接必须是**题目级别 ↔ 知识点级别**：错题文件指向具体知识点 block，知识点索引反链到具体错题 block
- block id 约定：
  - 知识点统一使用 `^kp-知识点ID`
  - 错题统一使用 `^wrong-YYYY-MM-DD-NN`
- 如果不知道知识点在哪章节，先记录为"待确认"，后续补充
- 错误原因勾选：知识点遗漏、理解偏差、粗心大意、审题不清、计算错误

## 使用实时网络搜索补充参考文档

在记录错题时，如果遇到以下情况，应使用当前环境可用的浏览或搜索工具获取权威参考：

### 需要搜索的场景

1. **知识点不确定**：无法确定题目涉及的具体知识点章节
2. **答案需要验证**：正确答案存在疑问，需要确认
3. **概念不清**：对某个专业概念理解模糊
4. **新技术/新趋势**：涉及当年新增的考点
5. **需要补充解析**：现有知识无法完整解析题目

### 搜索策略

- 搜索关键词：软考、系统架构设计师、题目相关核心概念
- 优先查找：官方教材、历年真题解析、培训机构文章
- 技术概念：搜索权威技术文档、RFC标准、官方文档

### 执行流程

1. **分析题目关键词**：提取2-4个核心关键词
2. **执行网络搜索**：使用当前可用的 web/search 工具搜索
3. **筛选可靠来源**：优先选择官方文档、知名教程、权威解释
4. **记录参考文档**：在解析中添加参考链接

### 参考文档格式

在错题表格的"解析"部分添加参考文档：

```markdown
| 解析 | 详细解析内容

**参考文档**：
- [文档标题](URL)
- [文档标题2](URL2) |
```

### 示例

**搜索场景**：用户提交了一道关于"零信任架构"的错题

**执行搜索**：
```
查询：零信任架构 软考 系统架构设计师
```

**获取结果后更新错题**：

```markdown
| 解析 | 零信任架构是一种安全模型，核心原则是"永不信任，始终验证"...

**参考文档**：
- [零信任安全架构详解 - 阿里云](https://help.aliyun.com/...)
- [NIST零信任架构标准](https://csrc.nist.gov/...) |
```

### 注意事项

- 搜索词简洁准确，避免过长
- 每道题至少记录1-2个参考文档链接
- 优先选择中文资源，其次是英文官方文档
- 如果搜索结果无有效参考，记录"待补充"
