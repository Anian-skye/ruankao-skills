# ruankao-init

本技能是开源软考技能包的入口。它只做初始化、资料检查和下一步路由，不直接承担批改、抽问或论文训练。

## 目标目录

默认在当前 workspace 下建立：

```text
软考/
├── 资料/
│   ├── 教材/
│   ├── 综合题/
│   ├── 案例题/
│   ├── 论文范文/
│   ├── 论文知识/
│   └── 项目背景/
└── 输出/
    ├── 知识点索引.md
    ├── 错题本/
    ├── 案例冲刺/
    ├── 案例2sigma/
    ├── 测试/
    └── 论文素材/
```

如果用户已有自己的目录，尊重现有目录，只补缺失项，不移动大批文件。

## 初始化流程

### 1. 建目录

创建缺失目录。不要删除用户已有资料。

### 2. 生成项目背景卡

若 `软考/资料/项目背景/项目背景卡.md` 不存在，按 [references/project-background-template.md](references/project-background-template.md) 创建空模板。

要求：

- 不写任何示例公司、项目名、金额、团队规模。
- 不编造用户经历。
- 只保留字段，让用户自行填写。

### 3. 检查上传资料

检查并输出最小清单：

- 综合题资料：选择题/综合知识教材、题库、真题或错题来源。
- 案例题资料：案例教材、案例真题解析、案例模拟题或题库。
- 论文资料：论文写作教材、论文范文、论文知识笔记。
- 项目背景：项目背景卡是否已填写。

资料不足时不要中断，按可用资料给下一步建议。

### 4. 建基础索引

若 `软考/输出/知识点索引.md` 不存在，创建一个空索引：

```markdown
# 软考知识点索引

> 由用户上传资料逐步沉淀。错题记录和案例训练会逐步补充 `^kp-*` block id。

## 综合题

## 案例题

## 论文
```

### 5. 下一步路由

根据用户目标给出一个下一步：

- 综合题/选择题：用 `ruankao-wrong-note` 记录错题，再用错题复习技能复盘。
- 案例题抽问：用 `ruankao-case-2sigma-review` 或 `ruankao-case-sprint-10`。
- 案例模拟题：用 `ruankao-case-mock-generator`。
- 案例/简答评分：用 `ruankao-answer-grader`。
- 论文背诵和迁移：用 `ruankao-thesis-2sigma`。
- 论文论点素材：用 `ruankao-thesis-argument-prep`。
- 完整论文批改：用 `ruankao-thesis-reviewer`。
- 不确定：用 `ruankao-coach-agent`，由教练判断下一步。

## 输出格式

初始化完成后，用短格式：

```markdown
初始化完成：
- 已创建：……
- 已检测到：……
- 待上传/待填写：……
- 下一步建议：……
```

## 禁止事项

- 不要内置或生成任何私人项目背景。
- 不要把示例资料当成用户资料。
- 不要批量移动或重命名用户已有资料。
- 不要默认下载第三方教材或真题。
- 不要在资料不完整时假装已经可完整训练；要说明当前能做哪一部分。
