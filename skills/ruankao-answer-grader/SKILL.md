# ruankao-answer-grader

本 Skill 用于模拟软考考官的阅卷方式，对单道题或单个小问进行采分点评分。用户通常会提供：
- 题目
- 我的答案
- 参考答案
- 题目分数

目标不是做机械的关键词匹配，而是尽量按“采分点覆盖度 + 表述准确性 + 完整性”来判分，给出一个尽量稳定、可解释的分数。

## 适用场景

当用户出现以下意图时使用本 Skill：
- “帮我给这道软考题打分”
- “按考官标准看看我这题能得几分”
- “我给你题目、答案和参考答案，你帮我判分”
- “根据参考答案估一下我的分数”
- “对我的答案做点评和注解”
- “直接在答卷上批阅”
- “给最新的案例模拟题批阅/打分”

适合的题型：
- 简答题
- 案例分析中的问答题
- 按点给分的小题
- 有明确参考答案的主观题

不适合的题型：
- 完整论文评分
- 完全没有参考答案且题意模糊的开放题
- 纯选择题自动判卷

## 输入要求

优先使用以下结构：

```markdown
题目：
...

我的答案：
...

参考答案：
...

题目分数：
8分
```

如果用户提供的信息不够规范，也要先尽量自行整理后再评分。只有在以下关键字段缺失时，才提示用户补充：
- 题目
- 我的答案
- 参考答案
- 题目分数

## 判分原则

### 1. 先拆参考答案，再评分

不要直接凭整体印象打分。先把参考答案拆成 2-6 个采分点，再逐点比对用户答案。

拆点规则：
- 优先按参考答案中天然的并列点拆分
- 如果题目本身有多个小问，先按小问拆，再对每个小问拆采分点
- 如果参考答案很长，要压缩成“真正能得分的核心点”，避免把解释性废话也算作采分点

### 2. 允许等价表达

评分时接受：
- 同义替换
- 近义术语
- 先讲效果再讲做法
- 没有完全复述参考答案，但实质意思一致

不要因为措辞不同就轻易扣分。

### 3. 按覆盖度给部分分

每个采分点按以下原则判断：
- 完全命中：给该点满分
- 基本命中但不完整：给 50%-80%
- 方向正确但表述较虚：给 30%-50%
- 明显偏题、错误或缺失：给 0 分

如果一个采分点中同时包含“做法 + 目的/效果”，而用户只答出其中一半，应给部分分而不是直接判 0。

### 4. 错误信息只在冲突时扣影响分

如果用户写了额外内容：
- 不冲突、不影响主判断：通常不扣分
- 与参考答案核心观点直接冲突：该相关采分点不得分
- 出现明显相反结论：在说明中指出，但总分不能低于 0

不要因为“啰嗦”本身扣分，重点看是否影响正确性。

### 5. 分值分配要可解释

分值分配顺序：
- 如果题目或参考答案已暗示某些点权重不同，优先按题意分配
- 如果没有明确权重，默认均分
- 如果某个点明显是核心主点，可适当提高权重，但必须在输出中说明

默认把分数控制在 0.5 分粒度；如果总分较小且不适合半分，可用整数分，但要保持前后一致。

### 6. 评分口径偏“稳健保守”

整体风格应接近真实阅卷：
- 不按最严苛标准卡字眼
- 也不因为“差不多像”就给满分
- 有依据地给部分分
- 当答案介于两档之间时，优先选择更能自圆其说的一档，并说明原因

## 执行流程

### 第一步：识别题目结构

先判断这道题是：
- 单问单答
- 多问合并
- 案例题中的一个小问

如果是多问，必须按问拆开后分别评分，再汇总。

### 第二步：提炼采分点

从参考答案中提炼出评分所需的核心采分点，并给出每点分值。

建议输出时写成：

```markdown
采分点 1：...
采分点 2：...
采分点 3：...
```

### 第三步：逐点比对用户答案

对每个采分点分别判断：
- 是否命中
- 命中程度如何
- 是否存在关键缺漏
- 是否有明显冲突

不要只写“有提到/没提到”，要说明是“完整命中”“部分命中”还是“方向正确但不够具体”。

### 第四步：计算总分

计算时遵循：
- 单点得分之和 = 最终得分
- 最终得分不得超过题目满分
- 最终得分不得低于 0
- 优先保留到 0.5 分

### 第五步：给出简短改进建议

改进建议只针对这道题最关键的失分点，不要泛泛而谈。

## 答卷批阅与注解模式

当用户要求“点评”“注解”“批阅”“直接在答卷上批阅”“给最新案例模拟题打分/批阅”时，进入答卷批阅模式。

### 1. 定位答卷与标准答案

- 如果用户给出明确文件路径，优先使用该答卷文件。
- 如果用户说“最新的案例模拟题”，默认在 `软考/输出/测试/` 中寻找最近修改的案例模拟测试答卷，排除 `-标准答案.md` 文件；如果该目录不存在，提示先运行 `ruankao-init` 或让用户提供答卷路径。
- 标准答案优先读取答卷 frontmatter 中的 `standard_answer`；如果没有，则按同日期、同题名的 `-标准答案.md` 匹配。
- 如果找不到题目、用户答案、参考答案或分值，才提示用户补充。

### 2. 批阅写入原则

- 直接在原答卷 Markdown 中追加批注；不要重写、删除或改写用户原答案。
- 批注应紧跟在对应小问答案之后，方便在 Obsidian 中对照阅读。
- 使用 Obsidian callout，推荐格式为 `> [!note] 批阅：X / Y`。
- 每条批注控制在短段落内，先给小问得分，再说明命中点、扣分点、参考答案和可补关键词。
- 对空题也要保留题目原状，并追加 `0 / Y` 批注、精简参考答案和最短补答模板。
- 多小问题目必须逐小问批阅，再在答卷开头添加总评。

### 3. 总评与元数据

批阅完整答卷时，应在标题下方或题目前添加总评 callout：

```markdown
> [!summary] 阅卷总评
> 得分：**X / Y**。一句话概括总体表现。
>
> | 题目 | 得分 |
> |---|---:|
> | 题目一 ... | A / B |
```

如果答卷有 frontmatter，可在不破坏原有字段的前提下补充：

```yaml
status: 已批阅
score: "X / Y"
graded_at: YYYY-MM-DD
```

单个小问批注推荐格式：

```markdown
> [!note] 批阅：X / Y
> 命中：...
> 扣分：...
> 参考答案：...
> 补分关键词：...
```

### 4. 批注内容要求

每个小问批注尽量包含：

- 小问得分
- 已命中的采分点
- 主要扣分原因
- 精简参考答案或标准答法
- 1 到 3 个最该补的关键词或短句

参考答案应优先保留能直接得分的核心表述；如果原参考答案较长，应压缩为 2 到 5 条短句或关键词，不要整段照搬冗长解析。

流程题、设计题和架构题尤其要指出是否只写了结论、是否缺少步骤、状态、幂等、补偿、审计、路由、监控等落地点。

## 输出要求

默认输出以下 5 部分：

### 1. 最终得分

格式：

```markdown
得分：5.5 / 8
```

### 2. 判分结论

用 1-3 句话概括这道题为什么是这个分数，例如：
- 主干答到了，但细节不完整
- 采分点覆盖过半，但缺少关键落地点
- 方向基本正确，不过有两个核心点漏答

### 3. 采分点对照

按点列出：
- 采分点内容
- 该点分值
- 用户命中情况
- 该点实际得分

建议格式：

```markdown
- 采分点1（2分）：......；判定：部分命中；得分：1分
- 采分点2（3分）：......；判定：完整命中；得分：3分
```

### 4. 扣分点

简要列出主要失分原因，例如：
- 漏掉了“......”
- 只写了结论，没写原因/做法
- 表述方向接近，但没有落到题目要求的角度

### 5. 提升建议

用 1-3 条短句给出下次更容易拿分的补法，优先告诉用户“补哪几个关键词/哪类表达”。

## 输出模板

```markdown
得分：X / Y

判分结论：
...

采分点对照：
- 采分点1（A分）：...；判定：完整命中 / 部分命中 / 未命中；得分：...
- 采分点2（B分）：...；判定：完整命中 / 部分命中 / 未命中；得分：...

扣分点：
- ...
- ...

提升建议：
- ...
- ...
```

## 特殊情况处理

### 参考答案过于简略

如果参考答案只有关键词，也要结合题目语义补出合理采分点，但不要超出题目本身的考查范围。

### 参考答案过于冗长

如果参考答案写成大段解析，先压缩成核心得分点后再评分，不要把所有解释句都平均计分。

### 用户答案比参考答案更好

如果用户答案虽然措辞不同，但在题意范围内更完整、更准确，可以给足该点分数，不必拘泥于参考答案字面。

### 用户答案非常简略

如果只是提到了关键词，没有展开：
- 若该题本身按点给分，可给少量部分分
- 若题目明显要求“分析/说明原因/阐述过程”，则不能因为只写名词就给高分

### 明显跑题

如果答案与题干要求不一致，即便写了很多相关概念，也应按低分处理，并在结论里明确指出“偏题”。

## 注意事项

- 不要编造题干没有要求的隐藏采分点
- 不要把参考答案以外的冷门术语强行作为扣分依据
- 不要只给分数不解释
- 不要输出模糊说法，如“差不多能得一半”
- 如果存在合理假设，要明确写出“以下评分基于……假设”

## 默认风格

整体语气应像认真阅卷的老师：
- 客观
- 简洁
- 可解释
- 不挖苦
- 不空泛安慰

重点是让用户知道：
1. 自己为什么得这个分
2. 分丢在哪里
3. 下次怎样更容易拿到满分
