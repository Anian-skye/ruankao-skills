# 软考案例出题风格参考

本文件不是教材摘录，而是通用命题约束，供生成模拟题时快速对齐软考案例分析风格。具体资料由用户上传。

## 落盘约定

每次生成模拟题时，结果应固定保存到 `软考/输出/测试/` 目录，并拆成两个文件：

- `案例模拟测试-YYYY-MM-DD[-主题].md`
- `案例模拟测试-YYYY-MM-DD[-主题]-标准答案.md`

题目文件只放测试卷本体，答案文件只放参考答案，不要混写。

两个文件之间应互相链接。如果用户已有测试文件，沿用其 frontmatter 和链接方式；否则使用最小字段：`title/date/tags/standard_answer/status`。

## 参考来源

- 教材索引：`软考/输出/案例冲刺/题库.json`
- 真题索引：`软考/输出/案例冲刺/真题索引.json`
- 用户上传的案例教材：`软考/资料/教材/` 或 `软考/资料/案例题/`
- 用户上传的真题/解析：`软考/资料/案例题/`

## 真题最稳定的结构信号

### 1. 先业务场景，后问题

真题一般不会直接问“什么是某某概念”，而是先给一个业务背景，再附带：

- 功能列表
- 性能或安全约束
- 故障现象
- 扩展需求
- 接入要求

这些信息决定了后续每个小问的落点。

### 2. 一道大题通常包含 3 到 4 个小问

小问之间常见递进关系：

1. 先补全、识别、判断
2. 再说明原理、关系或差异
3. 最后做设计、选型、优化或综合分析

### 3. 问法偏“采分点直给”

真题更常见的写法是：

- `补充完善空(1)~(6)处内容`
- `指出有哪些参与者`
- `分别说明`
- `比较两者差异`
- `写出至少 3 种解决方案并说明适用场景`
- `请用 100/200 字以内说明`

不常见的写法是：

- `谈谈你的理解`
- `总结常见考点`
- `给出学习建议`

### 4. 业务名词会伪装知识点

即使考的是 UML、质量属性、缓存设计，题干也常借助具体行业场景表达，例如：

- 教学管理
- 医疗预约
- 图书馆管理
- 短视频平台
- 煤矿安全预警
- 智能家居

生成模拟题时要先写业务，再把知识点埋进问题里。

## 各类题目的典型问法

### 系统设计与建模

高频落点：

- `DFD` 补全加工、数据流、数据存储，并追问数据平衡原则
- `E-R` 图补实体、联系、属性
- `用例图` 识别参与者、关系类型、系统边界
- `类图/顺序图/活动图` 识别元素并说明作用

命题特征：

- 经常从“需求描述 -> 图形元素补全”切入
- 经常要求简述某种建模方法在分析或设计阶段的作用

### 系统架构设计与评估

高频落点：

- 质量属性识别与场景化
- 开发期/运行期质量属性区分
- 架构风格、模式、战术的场景匹配
- 风险点、敏感点、权衡点

命题特征：

- 先给出多个约束，再要求选风格或战术
- 答案必须和题干约束逐一对应，不能只背概念

### 数据库与缓存

高频落点：

- `RDB`、`AOF`、主从复制、Cluster、事务
- 缓存穿透、击穿、雪崩、热 Key、内存淘汰
- 数据分区、分库分表、读写分离

命题特征：

- 先比较原理差异
- 再分析故障原因
- 最后要求给出多种措施并说明适用场景

### Web 与综合架构

高频落点：

- 首页推荐、热点流量、异步上传、消息队列、缓存预热
- 分片压力、链路延迟、重复推荐、上传超时

命题特征：

- 更偏故障分析和架构优化
- 需要把组件职责和数据流说完整

### 系统安全

高频落点：

- 对称/非对称加密
- 数字证书
- 数字签名
- 访问控制
- 审计日志

命题特征：

- 往往借助合规或敏感数据场景切题
- 小问常要求把机制和业务动作一一对应

## 标准答案的常见样子

标准答案一般不是长篇议论文，而是：

- 先给结论
- 再按采分点分条
- 必要时补一句原因

例如：

- 质量属性题：先列出属性，再按六要素展开
- 设计模式题：先写模式名称和类型，再写适用场景和原因
- 故障优化题：先写原因，再写措施，再写适用场景或收益

## 生成时的禁区

以下内容一出现，通常就不够像软考真题：

- 把知识点名称直接放进题目标题里反复提示
- 让用户“从学习角度总结”
- 问题过于开放，缺少具体答题边界
- 一整道题全是泛泛而谈，没有补全、比较、判断或设计动作
- 标准答案没有采分点，只是一段解释

## 推荐的出题节奏

### 单题模式

1. 写 1 段业务背景
2. 列 4 到 6 条功能或约束
3. 写 3 个小问
4. 给出分值与标准答案

### 套题模式

优先覆盖 3 到 5 个不同方向，不要 5 道题都考同一种能力。

推荐组合：

1. 建模题
2. 架构设计题
3. 数据库/缓存题
4. 设计模式或安全题
5. Web 综合优化题
